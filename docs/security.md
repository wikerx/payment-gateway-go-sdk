# 安全说明

## 服务端使用

SDK 包含商户 API 私钥、平台请求公钥、商户响应私钥和可能的支付敏感数据，只能部署在商户服务端。

不要在以下环境使用：

- 浏览器
- 移动端 App
- 桌面客户端
- 小程序前端
- 任何会暴露密钥或卡数据的运行环境

## 日志

`payment.gateway.debug-raw-log-enabled=true` 会输出沙盒联调需要的请求明文、请求密文、响应原始密文参数、响应参数拆分、响应明文等信息。生产环境建议关闭。

SDK 会对卡号、CVC、邮箱、手机号、密钥、Authorization 等字段做基础脱敏。商户自己的业务日志、错误采集和链路追踪仍需继续做敏感字段治理。

## JWT 防重放

JWT `jti` 每次请求都由 SDK 自动生成，当前由业务前缀、纳秒时间和 16 字节安全随机数组成。不要把商户订单号、交易号、退款号固定作为 `jti`，否则重复查询、重复取消或重试同一业务请求可能触发网关防重放。

业务幂等应使用请求体内的订单号、退款号、交易号等业务字段完成。

## 金额和币种

金额字段使用字符串传输，不使用 `float64` 表达金额。商户系统落库、展示、对账时应按币种精度处理。

## Webhook

Webhook 验证器会校验响应外壳和 `livemode`，并解密 `data`。商户处理回调时仍需：

- 做回调幂等；
- 只允许合法状态流转；
- 不允许终态被覆盖；
- 按订单号或交易号落库；
- 对异常回调保留原文用于排查。
